## Creating Screenshots - Step by Step

### Method 1: Direct from App (Authentic)

**Step 1: Prepare Demo Data**
1. Create sample invoices with realistic data
2. Add diverse customers
3. Populate inventory items
4. Ensure analytics show good numbers

**Step 2: Capture Screenshots**
1. Use Android emulator or real device
2. Set device to 1080x2400 resolution
3. Hide status bar icons (demo mode)
4. Capture each required screen
5. Save as PNG for quality

**Step 3: Add Captions in Design Tool**
1. Open in Figma/Canva/Photoshop
2. Extend canvas to 1080x2400 if cropped
3. Add 720px caption area at bottom
4. Apply gradient background
5. Add text with specified styles
6. Export as PNG

### Method 2: Using Device Mockup (Premium Look)

**Step 1: Capture Raw Screenshots**
- Same as Method 1

**Step 2: Create in Figma/Canva**
1. Create 1080x2400 canvas
2. Add caption area first (bottom 720px)
3. Add device mockup (top 1680px area)
4. Place screenshot inside device frame
5. Add subtle shadow around device
6. Export as PNG

**Mockup Resources:**
- Figma: "Device Mockup" plugin
- Canva: Search "phone mockup"
- Facebook Design Resources (free)

### Method 3: Quick & Easy (Canva)

1. Open Canva
2. Create custom size: 1080 x 2400px
3. Use "Phone Mockup" template
4. Upload your screenshots
5. Add text at bottom with gradient background
6. Download as PNG

## Screenshot Testing Checklist

Before uploading to Play Store:

**Visual Quality:**
- [ ] High resolution (1080x2400)
- [ ] No pixelation or blurriness
- [ ] Text is readable at thumbnail size
- [ ] Colors are vibrant and on-brand
- [ ] Consistent design across all screenshots

**Content:**
- [ ] No personal or real customer data
- [ ] Demo data looks realistic
- [ ] No placeholder text (Lorem ipsum)
- [ ] No debug/developer mode visible
- [ ] Status bar hidden or clean

**Captions:**
- [ ] No spelling errors
- [ ] Clear, concise messaging
- [ ] Highlights unique value
- [ ] Consistent tone across all
- [ ] Call-to-action in final screenshot

**Technical:**
- [ ] Correct dimensions (1080x2400)
- [ ] File size under 8MB
- [ ] PNG or JPEG format
- [ ] File names descriptive (screenshot_1_dashboard.png)

## A/B Testing Strategy

Create two sets of screenshots:

### Set A: Feature-Focused
- Emphasize what the app does
- Technical capabilities
- "CREATE INVOICES", "TRACK PAYMENTS", etc.

### Set B: Benefit-Focused

## Final Pro Tips

1. **First Impression Matters:** Screenshot #1 is most important
2. **Tell a Story:** Order screenshots to show user journey
3. **Less Text is More:** Captions should be scannable in 2 seconds
4. **Test on Mobile:** View in Play Store on actual device
5. **Update Regularly:** Refresh every 3-6 months with new features
6. **Localize if Possible:** Different captions for different markets
7. **Consistency:** Use same caption design across all screenshots
8. **Contrast:** Ensure text is readable against background
9. **Action Verbs:** Use strong verbs (Create, Track, Manage, Build)
10. **Numbers:** Include stats when possible ("Save 5 hours", "10,000 users")
